# AGENTS.md

## Project rules
- Интерфейс приложения должен быть на русском.
- Изучаемый язык — французский.
- Не удаляй существующие упражнения.
- После изменений обязательно проверяй, что проект собирается через `npm run build`.
- Новые компоненты делай строго типизированными на TypeScript.

## UX and product requirements
- Ежедневный урок должен быть разделён на понятные модули с отдельными визуальными блоками и индикатором прогресса.
- Минимальные модули ежедневного урока:
  1. Новые слова
  2. Тренировка новых слов
  3. Повторение не до конца выученных слов
  4. Закрепление
- Пользователь должен ясно видеть, в каком модуле он находится сейчас, сколько модулей осталось и какой прогресс по текущему дню.
- После завершения всех модулей ежедневного урока приложение должно показывать понятный экран состояния: `На сегодня заданий нет`.
- На экране завершения дня должны быть:
  - сообщение о завершении всех заданий на сегодня;
  - краткая статистика по пройденному уроку;
  - кнопка перехода в словарь или повторение сложных слов.
- Если пользователь уже знает слово, должна быть явная возможность пометить его как уже известное.
- Для этого добавь действие уровня карточки/экрана слова: `Уже знаю`.
- При выборе `Уже знаю` слово должно переходить в отдельную категорию известных/выученных слов и не должно снова показываться как новое слово в ежедневном уроке.
- В словаре должны быть отдельные разделы или фильтры:
  - Все слова
  - Изучаемые
  - Уже известные
  - Выученные
  - Сложные
- Интерфейс должен быть более интуитивным и удобным:
  - крупные и понятные кнопки;
  - очевидная навигация между главным экраном, уроком и словарём;
  - явные состояния ответа, прогресса и завершения;
  - меньше визуального шума;
  - более читаемая типографика;
  - понятные заголовки модулей и экранов.
- Главный экран должен ясно объяснять, что делать дальше: начать ежедневный урок, открыть словарь, посмотреть прогресс.
- Внутри урока пользователь не должен теряться: всегда должно быть видно название модуля, шаг внутри модуля и общий прогресс дня.

## Implementation notes
- Не ломай текущую архитектуру проекта без необходимости.
- Реализуй модульный ежедневный урок поверх существующей логики упражнений.
- Для статусов слов используй отдельные состояния, различающие как минимум:
  - new
  - learning
  - known
  - mastered
  - difficult
- Учитывай статус `known` при генерации ежедневного урока.
- Все новые UI-компоненты делай переиспользуемыми.

## Git workflow and version control
- Codex должен сохранять все изменения через git.
- После любых значимых изменений в коде необходимо создавать commit.
- Коммиты должны быть небольшими и логически разделёнными (feature, fix, refactor и т.д.).
- Используй понятные сообщения коммитов на английском, например:
  - `feat: add modular daily lesson system`
  - `fix: correct lesson progress calculation`
  - `ui: improve lesson module interface`
- После завершения задачи изменения должны быть закоммичены.
- Не переписывай историю git без необходимости.
- Не удаляй предыдущие версии кода.
- Сохраняй историю изменений, чтобы можно было откатиться к любой предыдущей версии.
- Если изменение крупное, делай несколько последовательных коммитов вместо одного большого.
- Всегда проверяй, что проект собирается (`npm run build`) перед финальным коммитом.
